# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Changed

- **`--wire-repo` / `-WireRepo` health check now propagates exit code by default.** Previously, a failing health check after `--wire-repo` installation printed a warning but always exited zero. The installer now exits with the health check's non-zero exit code so chained commands (`install.sh --wire-repo && next_thing`) correctly honor failure. Set `LLM_WIKI_HEALTH_CHECK_NONFATAL=1` to restore the old warn-only behavior. This is a **potential breaking change** for automation scripts that relied on the installer always succeeding. ([`install.sh`](install.sh), [`install.ps1`](install.ps1))

### Added

- **`wire_global_claude.py` heading-fallback fix.** When updating an older LLM Wiki section that lacks the `<!-- /llm-wiki -->` end-marker, section replacement now terminates on ANY markdown heading (`# ` or `## `), not just `## `. This prevents data loss of subsequent top-level headings in hand-edited `~/.claude/CLAUDE.md` files. ([`installers/wire_global_claude.py`](installers/wire_global_claude.py))
- **Regression tests for `wire_global_claude.py`.** Six permutation fixtures covering fresh insert, end-marker update, heading-fallback data-loss guard, append-to-existing, command copy idempotency, and dry-run zero-side-effects. ([`installers/tests/test_wire_global_claude.py`](installers/tests/test_wire_global_claude.py))
- **Updated agent-facing skill docs to teach `--wire-repo`.** The `g-kade` and `pokemon-benchmark` home skills now present `--wire-repo` as the primary one-command bootstrap path, replacing the legacy `llm_wiki_packet.ps1 init` flow. ([`skills/home/g-kade/SKILL.md`](skills/home/g-kade/SKILL.md), [`skills/home/pokemon-benchmark/SKILL.md`](skills/home/pokemon-benchmark/SKILL.md))
- **Optional Hugging Face integrations for local installs.** Added installer preflight detection for `hf`, optional `HF_TOKEN`-gated Hugging Face Hub MCP wiring into Claude Code and Factory, and an opt-in local TEI (`text-embeddings-inference`) docker service for embeddings. ([`installers/preflight.py`](installers/preflight.py), [`support/scripts/llm_wiki_memory_runtime.py`](support/scripts/llm_wiki_memory_runtime.py), [`docker-compose.yml`](docker-compose.yml), [`README.md`](README.md))
- **Regression tests for Hugging Face MCP wiring.** Runtime tests now verify both the skip path when `HF_TOKEN` is absent and the non-secret-persisting Claude/Factory wiring path when it is present. ([`tests/test_llm_wiki_memory_runtime.py`](tests/test_llm_wiki_memory_runtime.py))
- **Canonical roadmap and task tracking.** Added `ROADMAP.md` (milestones M1–M5 with success criteria and dependency graph) and `TODO.md` (granular checkbox tasks mapped to milestones). These close the loop on the SOTA memory systems comparison by turning identified gaps into an executable plan. ([`ROADMAP.md`](ROADMAP.md), [`TODO.md`](TODO.md))
- **Skill trigger classifier (M1).** Proactive skill suggestion at session start via `scripts/skill_trigger.py`. Supports three embedding backends (`keyword`, `tei`, `stub`), configurable threshold, and integration into `llm_wiki_agent_failure_capture.py`. ([`support/scripts/skill_index.py`](support/scripts/skill_index.py), [`support/scripts/skill_trigger.py`](support/scripts/skill_trigger.py), [`support/scripts/build_skill_index.py`](support/scripts/build_skill_index.py), [`tests/test_skill_trigger.py`](tests/test_skill_trigger.py))
- **Recency decay + graph traversal (M1 retrieval improvements).** Skill scores decay exponentially with age (`halflife_days`, default 30). Skills can declare `related_skills` edges (prerequisite, conflict, successor) in YAML frontmatter; the classifier traverses edges and emits neighborhoods. ([`support/scripts/skill_index.py`](support/scripts/skill_index.py), [`tests/test_skill_trigger.py`](tests/test_skill_trigger.py))
- **Reinforcement hook (M1 feedback loop).** After each session, the wrapper logs +1 or -1 feedback for the top suggested skill to `.llm-wiki/skill-pipeline/feedback.jsonl` based on success/failure. ([`support/scripts/llm_wiki_agent_failure_capture.py`](support/scripts/llm_wiki_agent_failure_capture.py))
- **Auto-reducer packets (M2).** `scripts/auto_reducer_watcher.py` records session start/end markers and drafts episodic summaries automatically. Includes `start`, `end`, `list`, `approve`, `reject`, `show` commands. Integrated into the agent wrapper. ([`support/scripts/auto_reducer_watcher.py`](support/scripts/auto_reducer_watcher.py), [`tests/test_auto_reducer_watcher.py`](tests/test_auto_reducer_watcher.py))
- **Taxonomy RFC (M3).** Published `docs/rfc-memory-taxonomy.md` formalizing the five-layer model, promotion state machine, retrieval contracts, MCP bindings, and comparison to MemGPT/Letta/WorldDB/Zep/Mem0. ([`docs/rfc-memory-taxonomy.md`](docs/rfc-memory-taxonomy.md))
- **Benchmark harness (M3).** Added `benchmarks/roadmap-m3-2026/harness.py` and README with protocol, metrics, and analysis script. ([`benchmarks/roadmap-m3-2026/harness.py`](benchmarks/roadmap-m3-2026/harness.py), [`benchmarks/roadmap-m3-2026/README.md`](benchmarks/roadmap-m3-2026/README.md))
- **Docker quickstart (M4).** Added `docker-compose.quickstart.yml` for one-command `docker compose up` with local-first defaults. ([`docker-compose.quickstart.yml`](docker-compose.quickstart.yml))
- **Unattended installer (M4).** Added `--unattended` to `install.sh` and `-Unattended` to `install.ps1` for zero-prompt CI/devcontainer installs. ([`install.sh`](install.sh), [`install.ps1`](install.ps1), [`tests/test_installer_flags.py`](tests/test_installer_flags.py))
- **Read-only memory dashboard (M5).** `scripts/dashboard_server.py` serves a responsive SPA at `/dashboard` with wiki search, skill list, BRV status, recent log entries, and Obsidian deep-links. Zero external dependencies (stdlib only). ([`support/scripts/dashboard_server.py`](support/scripts/dashboard_server.py), [`tests/test_dashboard_server.py`](tests/test_dashboard_server.py))
- **Preflight version detection.** `installers/preflight.py` now prints detected tool versions (Node, Docker, git, etc.) so users can see what's already installed. ([`installers/preflight.py`](installers/preflight.py))
- **Docker CI workflow.** Added `.github/workflows/docker-publish.yml` to build and push the packet image to GHCR on main branch pushes and version tags. ([`.github/workflows/docker-publish.yml`](.github/workflows/docker-publish.yml))
- **Ignore generated skill index artifact.** Added `.llm-wiki/skill-index.json` to `.gitignore` so rebuilds do not dirty the repo. ([`.gitignore`](.gitignore))
- **Automatic skill-index maintenance.** Setup and health-check flows now refresh `.llm-wiki/skill-index.json`, and runtime retrieval lazily rebuilds it when active skills, retired skills, feedback, or config changed. This removes a manual maintenance step for end users while keeping the index human-debuggable. ([`support/scripts/llm_wiki_memory_runtime.py`](support/scripts/llm_wiki_memory_runtime.py), [`support/scripts/skill_index.py`](support/scripts/skill_index.py), [`tests/test_skill_trigger.py`](tests/test_skill_trigger.py), [`tests/test_llm_wiki_memory_runtime.py`](tests/test_llm_wiki_memory_runtime.py))
- **Documentation QA refresh.** Corrected the bootstrap inventory and hosted-edge guidance, clarified automatic skill-index maintenance in operator docs, and added a formal ADR plus audit note for the derived-artifact lifecycle. ([`README.md`](README.md), [`QUICKSTART.md`](QUICKSTART.md), [`LLM_WIKI_MEMORY.md`](LLM_WIKI_MEMORY.md), [`deploy/cloudflare/README.md`](deploy/cloudflare/README.md), [`docs/rfc-memory-taxonomy.md`](docs/rfc-memory-taxonomy.md), [`docs/decisions/ADR-001-automatic-skill-index-maintenance.md`](docs/decisions/ADR-001-automatic-skill-index-maintenance.md), [`docs/documentation-audit-2026-04-22.md`](docs/documentation-audit-2026-04-22.md))
- **Windows hosted-installer long-path fix.** Shortened the PowerShell installer's temporary extraction root to avoid `Expand-Archive` failures on deep archive paths during hosted installs into repos and vaults. Added regression coverage for the short temp-root behavior. ([`install.ps1`](install.ps1), [`tests/test_installer_flags.py`](tests/test_installer_flags.py))
- **Wire-repo health check now skips GitVizz by default.** The final hosted-installer health check now skips GitVizz for `g-kade` / `--wire-repo` installs unless you explicitly opt back in, matching the default repo bootstrap behavior and avoiding false-red installs when GitVizz is intentionally not running yet. Also fixed the PowerShell-specific wrapper invocation to use the native `-SkipGitvizz` switch instead of the GNU-style `--skip-gitvizz` string. ([`install.ps1`](install.ps1), [`install.sh`](install.sh), [`tests/test_installer_flags.py`](tests/test_installer_flags.py))
- **Benchmark harness execution.** Ran stub-mode validation: 10 episodes per condition (Packet vs Baseline). Packet showed 100% completion rate vs 70% baseline, with 27.5% mean step reduction. Harness and analysis pipeline verified. ([`benchmarks/roadmap-m3-2026/harness.py`](benchmarks/roadmap-m3-2026/harness.py))
- **Negative-example filtering for skill retrieval.** Skill ranking now downranks retired skills and negatively-reviewed skills via persisted `penalties` in the skill index. Retired skills contribute a `0.5` penalty; each `-1` feedback verdict contributes `0.25`, capped so skills never disappear entirely. Also hardened index building to derive workspace-relative feedback paths from the output index location, not the active-skill directory shape. ([`support/scripts/skill_index.py`](support/scripts/skill_index.py), [`tests/test_skill_trigger.py`](tests/test_skill_trigger.py))
